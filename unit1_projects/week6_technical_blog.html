Variable Scope 1 June 2014

The ruby programming language supports four types of variable scope, local, global, instance, and class. Each variable type is declared by using a special character at the start of the variable name.

$ A global variable
@ An instance variable
[a-z] or _ A Local variable
[A-Z] A constant
@@ A class variable

Global variables are accessible anywhere inside the Ruby Program

$welcome = "Welcome to Ruby Essentials"

Global variables are strongly discouraged because they can be changed from anywhere inside the application. Which can make tracking bugs difficult.

Class Variables are variables that are shared amongst all instances or objects of the class. For example if one object changes the value of the class variable. All the other objects will now be using the new value of the variable

@@total = 0

Instance variables are similar to Class variables except that their values are local to specific instances of an object not to all objects/instances of the class. All objects share the same instance variable name but contain their own values of that variable name. For example if one object changes its value, the change only effects the object calling that variable.

@total = 10

Local variables are local to the code construct in which they are declared. For example a variable declared inside a method or function can only be used inside that method. That same variable cannot be accessed outside the method. This same rule also applies to loops.

loopcounter = 10 _LoopCounter = 20
